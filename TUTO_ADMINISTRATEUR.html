1. Se connecter à l'espace admin du site

Le site est accessible à l'adresse : https://sandrinepichla.github.io/amsp-website

Étapes :
  1. Aller sur le site
  2. Cliquer sur l'icône de personne en haut à droite → "Se connecter"
  3. Entrer votre adresse email et votre mot de passe administrateur
  4. Cliquer sur "Se connecter"

Une fois connecté, votre prénom apparaît en haut à droite. Vous verrez dans le menu un icône bouclier → "Gestion des membres".

⚠️ Si vous ne voyez pas "Gestion des membres", votre compte n'a pas les droits administrateur. Contactez le développeur.

2. Valider ou refuser une demande de membre

Lorsqu'une personne s'inscrit via la page "Rejoindre l'espace membre", vous recevez un email de notification et son compte est mis en attente.

Étapes :
  1. Connectez-vous à votre compte admin (voir section 1)
  2. Cliquer sur l'icône bouclier en haut à droite → "Gestion des membres"
  3. Dans la section "En attente", vous voyez les demandes à traiter
  4. Pour chaque demande, cliquer sur :
    • ✅ Approuver → la personne pourra se connecter à l'espace membre
    • ❌ Refuser → la personne ne pourra pas se connecter

Ce que voit le membre selon votre décision :

Décision Message affiché au membre lors de sa connexion
✅ Approuvé Il se connecte normalement à l'espace membre
⏳ En attente "Votre compte est en attente de validation par l'administrateur"
❌ Refusé "Votre demande d'adhésion a été refusée"
💡 Le badge orange sur "En attente" indique le nombre de demandes à traiter.

3. Promouvoir un membre en administrateur

Pour donner les droits administrateur à un membre (il pourra gérer les autres membres) :

Étapes :
  1. Aller dans "Gestion des membres"
  2. Dans la section "Membres actifs", cliquer sur le nom du membre pour dérouler sa fiche
  3. Cliquer sur "Promouvoir en admin" (bouton bleu)
  4. Une fenêtre de confirmation s'ouvre avec le nom et l'email du membre
  5. Cliquer sur "Confirmer" pour valider
⚠️ Un administrateur peut gérer tous les membres du site. Ne donnez ce rôle qu'à des personnes de confiance.

Pour retirer les droits admin, même procédure → bouton "Retirer les droits admin" (en rouge).

4. Modifier le contenu du site avec Sanity

Sanity est l'outil qui permet de modifier le contenu du site (textes, photos, actualités, disciplines…) sans toucher au code. Les modifications sont visibles sur le site en moins d'une minute.

Se connecter à Sanity

  1. Aller sur https://www.sanity.io
  2. Cliquer sur "Log in" en haut à droite
  3. Se connecter avec le compte Google ou email associé au projet AMSP
  4. Une fois connecté, accéder au studio AMSP

Ce que vous pouvez modifier

Rubrique dans Sanity Ce que vous pouvez faire
Disciplines Modifier descriptions, photos, noms des disciplines
Actualités Créer, modifier, supprimer des articles
Galerie Ajouter des albums photos, les rendre privés ou publics
Inscription Modifier le règlement intérieur, les infos de paiement, l'autorisation parentale…
Paramètres Modifier l'adresse, le téléphone, l'email de contact, les horaires

Comment modifier un contenu

Exemple : modifier le texte du règlement intérieur

  1. Dans Sanity Studio, cliquer sur "Inscription" dans le menu gauche
  2. Cliquer sur le document existant
  3. Trouver le champ "Règlement intérieur" et cliquer dessus
  4. Modifier le texte directement
  5. Cliquer sur "Publish" (bouton vert)

Exemple : ajouter une actualité

  1. Cliquer sur "Actualités" dans le menu gauche
  2. Cliquer sur le bouton "+" ou "New"
  3. Remplir le titre, le texte, ajouter une image
  4. Cliquer sur "Publish"

Exemple : ajouter des photos dans la galerie

  1. Cliquer sur "Galerie" dans le menu gauche
  2. Ouvrir l'album concerné (ou créer un nouveau avec "+")
  3. Cliquer sur le champ images → "Add item"
  4. Glisser-déposer ou sélectionner les photos depuis votre ordinateur
  5. Cliquer sur "Publish"
⚠️ Important : toujours cliquer sur "Publish" après une modification. Tant que vous ne publiez pas, les changements ne sont pas visibles sur le site.

5. Vérifier que les modifications sont bien en ligne

Après une modification dans Sanity

  1. Après avoir cliqué sur "Publish", attendre 30 secondes à 1 minute
  2. Aller sur le site : https://sandrinepichla.github.io/amsp-website
  3. Appuyer sur Ctrl + Maj + R (Windows) pour forcer le rechargement
  4. Naviguer vers la page modifiée pour vérifier
💡 Le contenu Sanity se met à jour automatiquement sur le site. Pas besoin de faire un redéploiement.

Si la mise à jour ne s'affiche pas

Vider complètement le cache du navigateur :

Sur Chrome :
  1. Appuyer sur Ctrl + Maj + Suppr
  2. Cocher "Images et fichiers en cache"
  3. Cliquer sur "Effacer les données"
  4. Recharger la page

Vérifier un déploiement de code (fait par le développeur)

  1. Aller sur https://github.com/SandrinePichla/amsp-website
  2. Cliquer sur l'onglet "Actions"
  3. La dernière ligne indique le statut du déploiement :
Icône Signification
✅ Cercle vert Déploiement réussi — le site est à jour
🟡 Cercle orange Déploiement en cours — attendre 2 à 3 minutes
❌ Croix rouge Erreur lors du déploiement — contacter le développeur

6. Modifier les emails automatiques avec EmailJS

EmailJS gère les emails envoyés automatiquement par le site :

  • Email reçu lorsqu'un membre remplit le formulaire d'inscription en ligne
  • Email reçu lorsqu'une personne crée un compte membre
  • Emails envoyés depuis le formulaire de contact

Se connecter à EmailJS

  1. Aller sur https://www.emailjs.com
  2. Cliquer sur "Sign In" en haut à droite
  3. Se connecter avec le compte associé au projet AMSP

Modifier le contenu d'un email

  1. Dans le menu gauche, cliquer sur "Email Templates"
  2. Cliquer sur le template à modifier
  3. Modifier le sujet (Subject) et le corps du message
  4. Cliquer sur "Save" pour enregistrer
⚠️ Les mots entre doubles accolades sont des variables automatiques — ne pas les modifier ni les supprimer :

{{nom}} → sera remplacé par le nom de la personne
{{email}} → sera remplacé par son adresse email
{{disciplines}} → sera remplacé par les disciplines choisies
{{prenom}}, {{tel_mobile}}, etc.

Tester un email avant de l'envoyer

Dans un template EmailJS, cliquer sur "Test It" pour envoyer un email de test et vérifier le rendu avant qu'il parte à de vrais destinataires.

Vérifier le quota d'emails

La version gratuite d'EmailJS permet 200 emails par mois.

Pour vérifier le nombre d'emails envoyés :
  1. Menu gauche → "Dashboard"
  2. Section "Email Sent" — affiche le compteur du mois en cours
💡 Si vous dépassez 200 emails/mois, les emails ne seront plus envoyés. Il faudra passer à un abonnement payant sur emailjs.com ou contacter le développeur.
